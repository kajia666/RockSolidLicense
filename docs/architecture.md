# 架构设计

## 总体目标

RockSolidLicense 面向“软件商业授权 + 在线鉴权 + 设备绑定 + 实时心跳”的典型场景，第一版优先保证：

- 可运行的端到端主链路
- 明确的领域边界
- 可以扩展到多节点部署的协议层
- 管理端与 SDK 之间有清晰分工

## 分层

### 管理后台层

- 管理员登录
- 产品与策略管理
- 卡密批量生成
- 在线会话监控

当前实现为原生 HTML + 管理 API，方便先打通业务流。后续可替换为 React/Vue 前端。

### 授权核心层

- 账号注册与登录
- 卡密兑换为授权窗口
- 设备绑定
- 会话签发和续约
- 心跳超时剔除

这一层是真正的授权中枢，后续建议拆成无状态服务，并接 Redis / MQ / PostgreSQL。

### SDK 接入层

- 客户端请求签名
- 机器码摘要
- 随机 nonce
- 协议封装

当前 SDK 先提供 C/C++ 接口与 Windows 加密实现，便于桌面软件快速接入。

## 鉴权流程

### 注册

1. SDK 生成时间戳、nonce、请求签名
2. 客户端调用 `/api/client/register`
3. 服务端验证签名和 nonce
4. 创建终端账号

### 充值

1. 用户输入卡密
2. 服务端校验卡密状态、策略、目标账号
3. 生成新的 `entitlement`
4. 卡密状态变更为 `redeemed`

当前实现采用“按授权窗口顺延”的方式叠加时长，不直接覆盖旧授权。

### 登录

1. 客户端上传账号、密码、机器码
2. 服务端找到当前生效的授权窗口
3. 校验设备绑定数量是否超限
4. 创建在线会话并下发：
   - `sessionToken`
   - `licenseToken`
   - 心跳配置

### 心跳

1. 客户端按照 `intervalSeconds` 上报心跳
2. 服务端续约 `expiresAt`
3. 若超时或机器码不匹配，则会话失效

## 安全建议

当前仓库已做：

- `scrypt` 密码哈希
- HMAC 请求签名
- 时间戳窗口校验
- nonce 防重放
- 设备指纹与授权绑定
- 会话续约与超时清理

上线前建议补充：

- 全链路 HTTPS / mTLS
- API 网关限流
- IP / 设备 / 账号风控
- SDK 秘钥分级轮换
- 审计日志异步写入
- 服务端签发公钥可验的授权令牌
- 重要策略变更二次确认

## 扩展方向

### 存储

- 本地开发: SQLite
- 生产建议: PostgreSQL + Redis

### 通信

- 当前: HTTP JSON
- 可扩展: TCP 二进制帧协议

### 部署

- 授权 API 无状态化
- 心跳写入 Redis
- 会话同步通过 Stream / MQ
- 管理台与授权服务分离部署
